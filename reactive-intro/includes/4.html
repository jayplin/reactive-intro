imperative

Imperative programing basically means
that the code gets executed in order
statement by statement.

If your debugger can clearly point to a statement in your code and it's also clear what might be the next line where it jumps to, then most likely you are using imperative programing. 


          let items = [];
          items.push(1);
          items.push(2);
          items.push(3);
          items.push(4);
          
          const newItems = items
            .filter(item => item % 2 === 0)
            .map(item => item + 0.5);
          
          newItems.forEach(item =>
            print(item)
          );
          // Output: 2.5
          // Output: 4.5
          
          items.push(5);
          items.push(6);
          items.push(7);
          items.push(8);
  1. we create an array
  2. add some values to it,
  3. then create a new array which is the modified version of the original one,
  4. console log the new array's items,
  5. and finally it adds a few more values to the original array.
